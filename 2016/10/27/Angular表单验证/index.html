Angular表单验证

  • 前端

常用的表单验证指令

  1. 必须项填写：

    <input type="text" required>

  2. 最短长度：

    <input type="text" ng-minlength="5">

    3.最大长度：

    <input type="text" ng-maxlength="10">

4.模式匹配：

<input type="text" ng-pattern="/[a-zA-Z]/">

5.email:

<input type="email" name="email" ng-model="user.email">

6.数字：

<input type="number" name="age" ng-model="user.age">

7.url:

<input type="url" name="url" ng-model="user.url>

上面的email,数字,url使用了HTML5验证。当符合验证要求的时候，才会将数据绑定到对应的ng-model上。

但是在HTML5支持不是很好的浏览器上，比如IE8 9中，这种HTML5验证方式就行不通了。解决方案是可以使用匹配模式，也可以使用自定义指令来复写或者重定义验证规则。

浏览器对表单有默认验证行为，当不符合验证信息的时候，会给出提示。如何屏蔽浏览器对表单的默认验证行为？在表单元素上加上novalidate标记。

表单的属性可以在其所属的$scope对象中访问到，而我们又可以访问$scope对象，因此JavaScript可以间接地访问DOM中的表单属性。借助这些属性，我们可以对表单做出实时响应。

可以使用formName.inputFieldName.property的格式访问这些属性。

  • $pristine: 没修改
  • $dirty: 修改过
  • $invalid: 验证失败
  • $valid: 验证成功

  • $error.required: 非空验证
